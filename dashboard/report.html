Executive Summary

  • V1 Scheduler Crisis: V1 scheduler deadlocks under concurrent load with CUDA graphs are blocking production deployments across Core Engine, Performance, and Installation SIGs, causing complete engine freezes at 0 tokens/s
  • GB200 Readiness Gap: FP8/FP4 kernel failures on SM120 GPUs and pipeline parallelism corruption issues threaten Q1's GB200 deployment targets for DeepSeek and other frontier models
  • Model Support Breakage: Critical models like DeepSeek-V3.2 and Qwen3-Next are generating invalid tokens or hanging indefinitely, blocking day-0 support commitments despite being roadmap priorities
  • Silent Failure Pattern: Multiple SIGs report silent failures where V1 engine drops requests, tests falsely pass, or models produce garbage output without errors, undermining reliability
  • Concurrent Request Instability: Race conditions in KV cache, request routing, and memory access appear across 5+ SIGs when handling concurrent requests, limiting vLLM to single-request debugging scenarios
  • FlashInfer Integration Risk: FlashInfer 0.4 breaks hybrid attention models with precision loss after 2nd turn, but the fix requires coordinating changes across attention backends, model implementations, and the sampler
